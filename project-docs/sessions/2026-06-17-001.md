### 1. Updated `src/App.tsx` — startDownload handler

Changed the `startDownload` callback from a placeholder that said "Keyword backend will be added in step 3" to a full implementation that:

- Sets appropriate status messages ("Searching and queuing results…")

- Includes all required dependencies in the useCallback dependency array

**Before:**

```typescript
} else {
    setStatus("ready");
    setStatusMsg("Keyword backend will be added in step 3");

```

**After:**

```typescript
} else if (mode === "keyword") {
    setStatusMsg("Searching and queuing results…");
    setBatchCurrent(0);
    try {
        await invoke("start_keyword_download", {
            query: keywordQuery.trim(),
            targetDir: savePath,
            maxHeight: keywordResolution,
            resultCount: keywordLimit,
        });
    } catch (err) {
        setStatus("error");
        setStatusMsg(`Keyword search failed: ${String(err)}`);
    }
}
```

### 2. Updated `src/App.tsx` — canDownload logic

**File:** [src/App.tsx#L574-L588](src/App.tsx#L574-L588)

Fixed the `canDownload` derived state to properly handle keyword mode:

- Replaced simple `batchFile !== ""` check with ternary logic for all three modes
- Keyword mode now checks that query is non-empty and app state is ready/done/error
- Download button is now enabled when conditions are met

**Before:**

```typescript
const canDownload =
    !isDownloading &&
    (mode === "single"
        ? (status === "ready" || status === "done") && selectedFmt !== ""
        : batchFile !== "" &&
          (status === "ready" || status === "done" || status === "error"));
```

**After:**

```typescript
const canDownload =
    !isDownloading &&
    (mode === "single"
        ? (status === "ready" || status === "done") && selectedFmt !== ""
        : mode === "batch"
          ? batchFile !== "" &&
            (status === "ready" || status === "done" || status === "error")
          : mode === "keyword"
            ? keywordQuery.trim() !== "" &&
              (status === "ready" || status === "done" || status === "error")
            : false);
```
