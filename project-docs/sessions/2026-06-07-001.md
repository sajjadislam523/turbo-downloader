## What was implemented and why

Sandboxed commands failed before executing with `No such file or directory`, even for `/bin/sh`, `/bin/bash`, and `/usr/bin/zsh`. Unsandboxed `/bin/sh` worked, and system inspection confirmed that `/usr/bin/zsh`, `/usr/bin/bash`, and `/bin/sh` all exist.

The likely cause was the global Codex `allowed_dirs` setting in `/home/kabila/.codex/config.toml`, which limited visible paths to project directories and hid system executables from the sandbox. The setting was commented out after creating a backup at `/home/kabila/.codex/config.toml.bak-2026-06-07-shell-sandbox`.
