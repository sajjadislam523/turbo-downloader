## What was implemented and why

The project instructions now require Codex to write every user idea, feature thought, workflow change, or rough plan into `.codex/plans`. If the idea is still unclear, Codex must append it to `.codex/plans/idea-inbox.md` with requirements and open questions. If it is ready for implementation, Codex must create or update a full feature plan before editing application code.

This closes the gap where an idea could exist only in chat history and then be unavailable after context compaction or a later session.
