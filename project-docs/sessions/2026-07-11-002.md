# Session 2026-07-11-002

## Problem
User reported: "the app crashes oftenly after running `npm run tauri dev` — the app opens, system says 'not responding', and terminal prints `Couldn't get key from code: AudioVolumeMute`."

## Investigation
1. TypeScript compiles clean (`tsc --noEmit`)
2. Rust compiles clean (`cargo check`)
3. Vite build succeeds (`npm run build`)
4. Frontend dependencies resolve
5. System deps present: `libwebkit2gtk-4.1-0`, `libgtk-3`, `yt-dlp 2026.07.04`
6. Confirmed `XDG_SESSION_TYPE=wayland` and `GDK_BACKEND=wayland` — running native Wayland
7. "Couldn't get key from code" is a debug `eprintln!` from `tao/src/platform_impl/linux/keyboard.rs:make_key_event()` — it fires for unmapped media keys but does **not** cause the crash
8. Root cause: GTK3 (which Tauri's `tao`/`wry` crates use) has known stability issues on native Wayland — window freezes, "not responding", event loop stalls. This is a well-documented issue across many Tauri v2 apps on Ubuntu/Fedora Wayland.

## Fix
**File:** `src-tauri/src/main.rs`

Added one line before app init:
```rust
std::env::set_var("GDK_BACKEND", "x11");
```

This forces GTK to use XWayland instead of native Wayland, which is fully stable with Tauri v2's stack.

## Verification
- `cargo check` passes
- User should run `npm run tauri dev` and confirm no "not responding" + no "Couldn't get key" messages
